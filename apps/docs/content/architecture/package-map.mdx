## Overview

Enterstellar is a **Turborepo monorepo** with 24+ packages under `packages/` and 3 apps under `apps/`. Packages are versioned independently (Babel model, managed with Changesets). `@enterstellar-ai/types` is always a peer dependency ‚Äî never a bundled one.

**Key structural rules (L15, enforced by CI):**
- All edges in the engine zone (everything except `react`, `devtools`, `cli`) are framework-free ‚Äî no `react`, `react-dom`, or `react-native` imports allowed.
- `@enterstellar-ai/types` is always a peer dep, never bundled.
- `@enterstellar-ai/react` is the only package that imports all engine packages as regular dependencies (RE1, RE2 locked).

## Package Descriptions

### Foundation

**`@enterstellar-ai/types`** ‚Äî The single source of truth for every type in the ecosystem. Zod schemas, TypeScript interfaces, type guards, and branded ID helpers. Every other package has `types` as a peer dep. Never bundled. Never circular.

**`@enterstellar-ai/contract-protocol`** ‚Äî JSON Schema Draft-07 files for non-TypeScript renderers. 8 schemas covering every data structure in the pipeline. No TypeScript source ‚Äî it is a specification package.

---

### Layer 1: Component Contract Registry

**`@enterstellar-ai/registry`** ‚Äî `defineComponent()` + `createRegistry()` + `mergeRegistries()`. Validates contracts at registration time (all 4 lifecycle states, Zod schema, token paths). Generates the compact Agent Manifest for LLM injection. Exposes `get()`, `list()`, `getManifest()`, `getSchema()`, and `register()`.

**`@enterstellar-ai/semantic-index`** ‚Äî Vector-based component retrieval. Embeds contracts at build time; embeds intents at runtime; returns top-K matches (~200 tokens). Three provider modes: `cloud` (Enterstellar Cloud default), `local` (offline-capable), `hybrid`.

**`@enterstellar-ai/normalizer`** ‚Äî Protocol adapters. Converts AG-UI events, A2UI blueprints, MCP widgets, or raw JSON into a unified `ComponentIntent`. Internal package ‚Äî not directly consumed by app developers.

**`@enterstellar-ai/forge`** ‚Äî Runtime contract generation for novel intents. LocalForge (template-based, client/edge) and CloudForge (LLM-based, Enterstellar Cloud). Output is always a `ComponentContract` ‚Äî never raw HTML (L1, L3). Every invocation emits a `ForgeSignal` (L12).

---

### Layer 2: UI Compiler

**`@enterstellar-ai/compiler`** ‚Äî The M1 moat. `createCompiler()` factory. 5-step pipeline: parse ‚Üí token check ‚Üí accessibility audit ‚Üí self-correction ‚Üí trace. `compiler.use()` accepts custom middleware steps. `compile()` returns `CompilationResult` with `status: 'pass' | 'corrected' | 'fail'`.

---

### Layer 3: Determinism Engine

**`@enterstellar-ai/cache`** ‚Äî LRU render cache. `createRenderCache()`. Cache key = `intentHash + componentName` (CA1 locked). Stores `CompilationResult` only ‚Äî not rendered React trees (CA2). `maxEntries: 1000`, `ttl: 3600` seconds by default. `cache.warmup()` is best-effort and never blocks (CA7).

---

### Layer 4: Lifecycle Manager

**`@enterstellar-ai/lifecycle`** ‚Äî 5-state FSM: `idle ‚Üí loading ‚Üí streaming ‚Üí ready ‚Üí error` (+ `empty`). Wraps every compiled component. Handles incremental prop arrival from the LLM stream. Internal package ‚Äî consumed exclusively by `@enterstellar-ai/react`.

---

### Layer 5: Observability

**`@enterstellar-ai/devtools`** ‚Äî Embedded `<EnterstellarDevTools />` panel. Zero bytes in production (`NODE_ENV=production` tree-shaking). Panels: Trace Timeline, Component Inspector, Validation Log. Toggle: `Ctrl+Shift+A`.

**`@enterstellar-ai/telemetry`** ‚Äî `ForgeSignal` collection and upload. Batches signals, queues offline, syncs on connectivity. IndexedDB queue by default (no synchronous writes). Enterprise opt-out via `queueStrategy: 'memory'`.

**`@enterstellar-ai/agent-sdk`** ‚Äî MCP server implementation. `createAgentSDK()` + `createMCPServer()`. 6 atomic tools + 1 composite `enterstellar_build_ui` tool.

---

### Layer 6: Adapter Layer

**`@enterstellar-ai/adapters`** ‚Äî Factories for all 4 adapter interfaces: `createAuthAdapter()`, `createDataAdapter()`, `createErrorAdapter()`, `createAnalyticsAdapter()`. All I/O methods async (AD2). All errors wrapped in `EnterstellarError` (AD5). No-op factories included for testing.

**`@enterstellar-ai/adapter-supabase`** ‚Äî `createSupabaseAuthAdapter({ client })` + `createSupabaseDataAdapter({ client })`. Delegates to `@enterstellar-ai/adapters` factories internally.

**`@enterstellar-ai/adapter-firebase`** ‚Äî `createFirebaseAuthAdapter({ app })` + `createFirebaseDataAdapter({ app })`. Same pattern as Supabase.

---

### Layer 7: Testing

**`@enterstellar-ai/test`** ‚Äî `createTestHarness({ registry })`. Deterministic mock pipeline: `harness.mock(intent, ComponentIntent)` ‚Üí `harness.resolve(intent)` ‚Üí `AgentTrace`. Vitest matchers via `enterstellarMatchers`. `harness.compileRaw()`, `harness.autoMock()`, `computeIntentCoverage()`, `saveFixtures()`, `loadFixtures()`.

---

### Tooling

**`@enterstellar-ai/cli`** ‚Äî `enterstellar init`, `enterstellar add component`, `enterstellar migrate`, `enterstellar review`. Manual arg parsing (no commander/yargs ‚Äî 4 commands don't justify the weight). `create-enterstellar-app` binary points here.

**`@enterstellar-ai/connection`** ‚Äî `createAgentConnection()` convenience factory. WebSocket/SSE/auto transport. Handles reconnection, keep-last on reconnect (P12 locked), and backpressure (P5 locked).

**`@enterstellar-ai/state`** ‚Äî `createEnterstellarStore()` (async). Persistence: `memory`, `local-storage`, `indexed-db`. Fixed keys: `'zones'`, `'traceIds'`, `'session'`. AES-GCM encryption. `snapshot()`, `restore()`, `registerMigration()`, `extend()`.

---

### Cloud

**`@enterstellar-ai/cloud`** ‚Äî Enterstellar Cloud SDK. IPU metering, trace aggregation, ForgeSignal upload. Internal package. Enterprise-only direct consumption.

**`@enterstellar-ai/global-index`** ‚Äî Federated registry discovery. Finds components across multiple published registries. Internal package. Powers the Global Semantic Index in cloud mode.

## Running the Monorepo

```bash
# Install all dependencies
pnpm install

# Build all packages
pnpm turbo run build

# Run all tests
pnpm turbo run test

# Test a specific package
pnpm --filter @enterstellar-ai/compiler test

# Type check everything
pnpm turbo run typecheck

# Lint everything
pnpm turbo run lint
```
